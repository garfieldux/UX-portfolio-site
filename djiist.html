My work in UX design

The Djiist

Objective: Add to existing news aggregator site to improve usability and social functions

"What is the problem?"

The Djiist is a news aggregator application that pulls information from Google Trends and displays it in a consumable format, using a photo and a short blurb for the content of each "news item". The client wants to expand the functionality of the application in the following ways:

  • Make sure the application is usable, easy to understand
  • Implement some kind of social media integration, show what people are saying about news online
  • Adapt the desktop version into a mobile application

"How do we solve it?"

In order to figure out how to implement the changes to the site and incorporate the new elements, I kept the following guidelines in mind:

  • Stories should be modular, self-contained
  • Let there be a hierarchy of information: start small, let user click to expand to learn more
  • Let user customize the experience
  • Use the inherent media to the advantage of the experience, let users see the story at a glance

Site Usability Audit

In order to see where the opportunities for improvement were on the existing site, I used the site extensively and exhaustively and found the following opportunities:

  • Links on stories go to external sites, there could be an opportunity to keep users on the site if content were outlined in a more detailed secondary page made visible with a click to the story
  • Social media element could enhance the experience of the site but it must remain secondary to the actual story content, that's why people come to the site
  • The site only updates every few hours; more frequent updates could keep users on the site, keen to stay up to date with the trends
